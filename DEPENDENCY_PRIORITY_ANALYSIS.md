## 二、优先级评估标准

### 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| **全链路压测相关性** | 40% | 是否直接影响影子库/影子表功能 |
| **使用率/流行度** | 30% | Python 项目中的使用广泛程度 |
| **实现复杂度** | 15% | 技术难度和预估工作量 |
| **业务价值** | 15% | 对用户业务场景的价值 |

### 优先级定义

| 优先级 | 评分 | 说明 | 建议周期 |
|--------|------|------|----------|
| **P0** | 85-100 | 核心功能，必须实现 | 立即 |
| **P1** | 70-84 | 重要功能，强烈推荐 | 2 周内 |
| **P2** | 55-69 | 有用功能，按需实现 | 1 月内 |
| **P3** | 40-54 | 边缘功能，可选实现 | 季度内 |
| **P4** | 0-39 | 低优先级，暂不实现 | 搁置 |

## 三、详细优先级分析

### 3.1 Web 框架 (Web Frameworks)

| 依赖 | 评分 | 优先级 | 评估理由 |
|------|------|--------|----------|
| `fastapi` | ✅ 已支持 | - | 已有完整插桩模块 |
| `flask` | ✅ 已支持 | - | P0 已完成 |
| `uvicorn` | ✅ 已支持 | - | ASGI 服务器 |
| `gunicorn` | 45 | P3 | WSGI 服务器，插桩价值有限 |

**建议**: gunicorn 插桩主要监控 Worker 指标，对影子库功能无直接影响。

---

### 3.2 数据库中间件 (Database Middleware) ⭐ 重点

| 依赖 | 评分 | 优先级 | 评估理由 |
|------|------|--------|----------|
| `sqlalchemy` | ✅ 已支持 | - | 已有基础插桩 |
| `pymysql` | ✅ 已支持 | - | MySQL 驱动 |
| `redis` | ✅ 已支持 | - | P0 已完成 |
| `flask-sqlalchemy` | 55 | P2 | Flask 集成，非独立中间件 |
| `elasticsearch7` | ✅ 已支持 | - | P1 已完成 |
| `pymongo` | 78 | **P1** | NoSQL 主流，影子库场景 |

**建议**: `pymongo` 是下一个数据库插桩重点，评分理由:
- 全链路压测相关性：40% × 8 = 32
- 使用率：30% × 9 = 27
- 实现复杂度：15% × 6 = 9
- 业务价值：15% × 7 = 10.5
- **总分：78.5**

---

### 3.3 缓存中间件 (Cache Middleware)

| 依赖 | 评分 | 优先级 | 评估理由 |

## 六、建议总结

### 6.1 核心建议

**下一步重点实现 (P1)**:
1. **MongoDB 插桩** - NoSQL 数据库影子库支持
2. **RabbitMQ 插桩** - 消息队列影子 Topic 支持
3. **LLM 可观测** - AI Agent 场景差异化优势

### 6.2 资源分配建议

| 优先级 | 建议资源 | 预期产出 |
|--------|----------|----------|
| P1 (3 个) | 60% | 覆盖 80% 主流中间件 |
| P2 (3 个) | 30% | 完善 HTTP 和 AI 场景 |
| P3/P4 | 10% | 按需响应 |

### 6.3 最终目标

实现后可覆盖:
- **数据库**: MySQL + MongoDB + Redis (3 种)
- **消息队列**: Kafka + RabbitMQ (2 种)
- **搜索引擎**: Elasticsearch (1 种)
- **Web 框架**: FastAPI + Flask (2 种)
- **AI 可观测**: Langfuse + Transformers (2 种)

**预计支持率**: 从 32% 提升至 **55%+**
